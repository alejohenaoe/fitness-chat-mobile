# Expo SDK 54 — read exact versioned docs
**https://docs.expo.dev/versions/v54.0.0/**

This is an Expo 54 / React Native 0.71 / React 18 project. No linter, no formatter, no test framework. `tsc` is available for typechecking (`npx tsc --noEmit`).

## Commands

| Command | What it does |
|---------|-------------|
| `npx expo start` | Dev server |
| `npx expo start --ios` | iOS simulator |
| `npx expo start --android` | Android emulator |
| `npx expo start --web` | Web dev |
| `npx tsc --noEmit` | Type-check only |

## Architecture

- **Entry**: `index.ts` → `import 'expo-router/entry'`
- **Routing**: expo-router file-based (`app/`), 2 groups: `(auth)` (login/register) and `(tabs)` (chat, progress, history, profile)
- **State**: Zustand single store (`src/stores/useAppStore.ts`) with auth, chat, progress, today-logs, UI slices
- **Server state**: TanStack React Query (hooks in `src/hooks/`)
- **API**: Axios instance (`src/services/api.ts`) with JWT interceptor — auto-attaches Bearer token, auto-refreshes on 401 via `/auth/token/refresh/`
- **Tokens**: `expo-secure-store` (access + refresh)
- **Forms**: `react-hook-form` + `zod` validation
- **Charts**: Custom SVG-based components (no charting library actively used)
- **Theme**: Dark only (`userInterfaceStyle: 'dark'`), shared `colors` + `layout` constants
- **Language**: Spanish (UI labels, date-fns locale)
- **UI patterns**: `react-native-modal` via RN `Modal`, RN `Animated` (no reanimated), `lucide-react-native` icons, `GestureHandlerRootView` wrapping root layout

## Key conventions

- API env var: `EXPO_PUBLIC_API_BASE_URL` in `.env` (use device IP, not `localhost`, when testing on physical device). `.env` is committed; `.env*.local` is gitignored for overrides.
- Use `StyleSheet.create` for styling; no CSS-in-JS libraries
- Zod schemas for form validation + API response parsing
- No class components, no Redux, no prop-drilling (Zustand + hooks)

## Gotchas

- No tests — no Jest config, no testing libraries installed. Do not write tests.
- No lint or format commands/setup. Do not add linter/formatter config unless asked.
- `expo-env.d.ts` is gitignored (auto-generated by Expo). If you add env vars, run `npx expo start` to regenerate it.
- **No `react-native-reanimated` or `@gorhom/bottom-sheet`** — these native modules are incompatible with Expo Go SDK 55. Animations use RN `Animated`; modals use RN `Modal`. Do not add native module dependencies without confirming Expo Go compatibility first.
- SVG charts via `react-native-svg` — `victory-native` and other skia-based chart libs won't work in Expo Go.
